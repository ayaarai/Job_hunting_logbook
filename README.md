# 企業選考ログ

転職活動の企業選考を記録・管理するWebアプリです。
`index.html` 1ファイルのみで動作し、ビルド不要です。

## 画面構成

| 画面 | 内容 |
|------|------|
| 一覧画面 | 統計カード・検索・フェーズフィルター・企業カード一覧 |
| 詳細画面（選考履歴タブ） | 選考履歴タイムライン・企業サマリー |
| 詳細画面（募集要項タブ） | 企業URL・添付資料URL・募集要項テキスト |
| 新規追加 | 企業情報入力フォーム |
| 編集画面 | 企業情報編集・削除 |

## 機能一覧

- 企業選考の記録を追加・編集・削除
- 選考フェーズをカラーバッジで視覚化
- 企業名・フェーズ・応募経路・媒体名でキーワード検索
- フェーズ別フィルタリング
- 統計カード（応募中 / 内定 / お見送り の件数を表示）
- **選考履歴タイムライン**（企業ごとに時系列で記録）
- **募集要項タブ**（企業URL・添付資料URL・募集要項テキストを企業ごとに保存）
- 応募経路が「エージェント」の場合のみエージェント名欄を表示
- 応募経路が「その他」の場合のみ媒体名欄を表示
- フェーズが「お見送り」の場合のみお見送り理由欄を表示
- `localStorage` によるデータ永続化（ページ再読み込み後も保持）
- **面接通知バナー**（面接日の前日・当日にアプリを開いたとき画面右側に通知表示）
- **JSONエクスポート**（全データをJSONファイルとしてダウンロード）
- **JSONインポート**（バックアップファイルの読み込み。既存データへの追加 or 置き換えを選択可能）
- 旧フォーマットのJSONインポート時も自動的にデータを補完（後方互換性あり）
- 初回起動時はモックデータ4件を自動表示

## 使い方

ビルド不要。`index.html` をブラウザで開くだけで動作します。

```bash
open index.html
```

またはVS Codeの Live Server 拡張などで開いてください。

### 基本的な操作フロー

1. ヘッダー右上「＋ 新規追加」から企業を登録
2. 一覧画面の企業カードをクリックで詳細画面へ
3. 詳細画面の **「選考履歴」タブ** から面接・結果などを時系列で追記
4. 詳細画面の **「募集要項」タブ** で企業URLや募集要項テキストを保存
5. 鉛筆アイコンで企業情報の編集

### データのバックアップと復元

| ボタン | 操作 |
|--------|------|
| ↑ インポート | JSONファイルを選択して読み込む |
| ↓ エクスポート | 全データを `job_logs_YYYY-MM-DD.json` としてダウンロード |

**インポート時の動作：**
- ファイル選択後、確認ダイアログが表示される
- **OK** → 既存データをすべて新しいデータで置き換え
- **キャンセル** → 既存データに追加（ID重複分はスキップ）

## データ構造

```json
{
  "id": "1710000000000",
  "company": "株式会社〇〇",
  "route": "エージェント",
  "agent": "リクルートエージェント",
  "media": "",
  "stage": "一次面接",
  "dismissalReason": "",
  "dismissalNote": "",
  "interviewDate": "2026-03-20",
  "interviewer": "田中部長",
  "memo": "全体的な所感メモ",
  "events": [
    { "id": "e1", "date": "2026-03-01", "type": "応募",    "memo": "" },
    { "id": "e2", "date": "2026-03-07", "type": "カジュアル面談", "memo": "雰囲気良好。" },
    { "id": "e3", "date": "2026-03-14", "type": "一次面接", "memo": "手応えあり。" }
  ],
  "details": {
    "jobDescription": "募集要項のテキスト...",
    "companyUrl": "https://example.com",
    "attachments": [
      "https://example.com/jd.pdf",
      "https://example.com/slide.pdf"
    ]
  },
  "createdAt": "2026-03-01T10:00:00Z"
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | 一意なID（`Date.now()` で自動生成） |
| `company` | string | 企業名（必須） |
| `route` | string | 応募経路 |
| `agent` | string | エージェント名（route が「エージェント」の場合） |
| `media` | string | 媒体名（route が「その他」の場合） |
| `stage` | string | 現在の選考フェーズ |
| `dismissalReason` | string | お見送り理由（stage が「お見送り」の場合） |
| `dismissalNote` | string | お見送り自由記述 |
| `interviewDate` | string | 次回面接日（YYYY-MM-DD） |
| `interviewer` | string | 面接官名 |
| `memo` | string | 企業全体のメモ |
| `events` | array | 選考履歴イベント一覧（タイムライン） |
| `events[].id` | string | イベントID |
| `events[].date` | string | イベント日（YYYY-MM-DD） |
| `events[].type` | string | イベント種別（応募 / フェーズ名 / その他メモ） |
| `events[].memo` | string | イベントのメモ |
| `details` | object | 募集要項情報（省略時は自動補完） |
| `details.jobDescription` | string | 募集要項テキスト |
| `details.companyUrl` | string | 企業URL |
| `details.attachments` | string[] | 添付資料URLの配列 |
| `createdAt` | string | 作成日時（ISO 8601） |

> **後方互換性：** `details` フィールドが存在しない旧フォーマットのデータをインポートまたは読み込んだ場合、`normalizeLog()` によって自動的にデフォルト値（空文字・空配列）が補完されます。

## 選択肢

### 応募経路

| 経路 | 追加入力欄 |
|------|-----------|
| エージェント | エージェント名 |
| 直接応募 | なし |
| LinkedIn | なし |
| リファラル | なし |
| その他 | 媒体名（Wantedly / Green / ビズリーチ 等） |

### 選考フェーズ・バッジカラー

| フェーズ | バッジカラー | 追加入力欄 |
|---------|------------|-----------|
| カジュアル面談 | グレー | なし |
| 書類選考中 | ブルー（薄） | なし |
| 一次面接 | ブルー（濃） | なし |
| 二次面接 | パープル | なし |
| 最終面接 | オレンジ | なし |
| 内定 | エメラルドグリーン | なし |
| お見送り | レッド | お見送り理由・自由記述 |

### お見送り理由（プリセット）

スキル不足 / カルチャーフィット / ポジションクローズ / 不明 / その他

## タイムライン機能

企業詳細画面では、選考の経緯を時系列で確認できます。

```
Amazon

  ● 2026/03/01   応募

  ● 2026/03/05   カジュアル面談
                 バックエンド構成の話で盛り上がった。

  ● 2026/03/10   一次面接
                 アルゴリズム問題。手応えあり。

  ● 2026/03/15   お見送り
                 理由: ポジションクローズ
```

- 左側の縦線＋丸ドットで時系列を視覚化
- 「選考を記録」ボタンからイベントを随時追記
- イベント追加時、ステージ系のイベントは現在フェーズを自動更新
- `お見送り` 記録時はお見送り理由もタイムラインに表示

## 変更履歴

### 面接通知バナー追加（2026-03-21）

**新機能**
- **面接通知バナー**を追加
  - `interviewDate`（次回面接日）が当日・前日の企業を自動検出して通知
  - 当日の面接は赤バッジ「本日」、前日は橙バッジ「明日」で表示
  - PCでは画面右上に固定表示（`lg` ブレークポイント以上）、スマホでは非表示
  - 個別の × ボタンで1件ずつ閉じるか、「すべて閉じる」で一括非表示
  - 閉じた通知のIDを `interview_notif_seen` キーで LocalStorage に保存し、再表示しない
  - お見送り・内定済みの企業は通知対象外
