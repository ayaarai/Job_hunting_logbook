# 企業選考ログ

転職活動の企業選考を記録・管理するWebアプリです。
`index.html` 1ファイルのみで動作し、ビルド不要です。

## 画面構成

| 画面 | 内容 |
|------|------|
| 一覧画面 | 統計カード・検索・フェーズフィルター・優先度フィルター・企業カード一覧 |
| 詳細画面（選考履歴タブ） | 選考履歴タイムライン（編集・削除対応）・企業サマリー・優先度セレクト |
| 詳細画面（募集要項タブ） | 企業URL・添付資料URL・募集要項テキスト |
| 詳細画面（メモタブ） | 企業ごとの自由メモ（リッチテキストエディタ） |
| 詳細画面（企業理解タブ） | 事業・課題特徴・技術取り組み・カルチャー・自分との接点・やりたいことの6セクションで企業理解を構造化記録 |
| 新規追加 | 企業情報入力フォーム |
| 編集画面 | 企業情報編集・削除 |

## 機能一覧

- 企業選考の記録を追加・編集・削除
- 選考フェーズをカラーバッジで視覚化
- 企業名・フェーズ・応募経路・媒体名でキーワード検索
- フェーズ別フィルタリング
- **優先度ラベル**（高/中/低の3段階。カラーバッジで色分け表示）
- 優先度フィルタリング（一覧画面から優先度で絞り込み可能）
- 統計カード（応募中 / 内定 / お見送り の件数を表示）
- **選考履歴タイムライン**（企業ごとに時系列で記録。カード形式で日付・種別・メモを視認しやすく表示）
- タイムライン項目の**閲覧モード**（通常はテキスト表示。改行・段落がそのまま見える。長文は「続きを読む」で展開）
- タイムライン項目の**編集モード**（鉛筆アイコンをクリックした時のみフォーム表示。入力に応じてテキストエリアが自動伸長）
- タイムライン項目の**削除**（インライン確認UI付き）
- **募集要項タブ**（企業URL・添付資料URL・募集要項テキストを企業ごとに保存）
- **メモタブ**（企業ごとの自由メモをリッチテキストエディタで編集・保存。太字・見出し・箇条書きに対応）
- **企業理解タブ**（6セクション構成で企業研究を構造化して記録）
  - 事業 / 課題・特徴 / 技術・取り組み / カルチャー / 自分との接点 / やりたいこと
  - 各セクションに概要（一言）・ポイント（複数行）・タグを入力
  - ポイントに★ハイライトをつけて重要項目をマーク
  - **面接直前モード**：ハイライト済みポイントのみをコンパクト表示
  - **志望動機変換**：各セクションの入力内容から志望動機ドラフトを自動生成してクリップボードにコピー可能
  - **MDを取り込む**：ローカルの `.md` ファイルを読み込み、`##` 見出しを基準に各セクションへ自動マッピング
  - セクションカードを折りたたんでスッキリ表示
- 応募経路が「エージェント」の場合のみエージェント名欄を表示
- 応募経路が「その他」の場合のみ媒体名欄を表示
- フェーズが「お見送り」の場合のみお見送り理由欄を表示
- `localStorage` によるデータ永続化（ページ再読み込み後も保持）
- **面接通知バナー**（面接日の前日・当日にアプリを開いたとき画面右側に通知表示）
- **JSONエクスポート**（全データをJSONファイルとしてダウンロード）
- **JSONインポート**（バックアップファイルの読み込み。既存データへの追加 or 置き換えを選択可能）
- 旧フォーマットのJSONインポート時も自動的にデータを補完（後方互換性あり）
- 初回起動時はモックデータ4件を自動表示

## 使い方

ビルド不要。`index.html` をブラウザで開くだけで動作します。

```bash
open index.html
```

またはVS Codeの Live Server 拡張などで開いてください。

### 基本的な操作フロー

1. ヘッダー右上「＋ 新規追加」から企業を登録
2. 一覧画面の企業カードをクリックで詳細画面へ
3. 詳細画面の **「選考履歴」タブ** から面接・結果などを時系列で追記
   - 各イベントの鉛筆アイコンで日付・種別・メモを後から編集
   - ゴミ箱アイコンで削除（確認UI表示）
4. 詳細画面の **「募集要項」タブ** で企業URLや募集要項テキストを保存
5. 詳細画面の **「メモ」タブ** で企業ごとの自由メモを記入・保存
6. 詳細画面の **「企業理解」タブ** で企業研究を6セクションに整理して保存
   - 各ポイントに★ハイライトを付けておくと「面接直前モード」で重要点だけ素早く確認
   - 「志望動機に変換」ボタンで入力内容をもとに志望動機ドラフトを生成
   - 「MDを取り込む」ボタンでローカルの `.md` ファイルから各セクションへ一括反映
7. 詳細画面サマリーカードの優先度セレクトで優先度を即時変更
8. 鉛筆アイコンで企業情報の編集

### データのバックアップと復元

| ボタン | 操作 |
|--------|------|
| ↑ インポート | JSONファイルを選択して読み込む |
| ↓ エクスポート | 全データを `job_logs_YYYY-MM-DD.json` としてダウンロード |

**インポート時の動作：**
- ファイル選択後、確認ダイアログが表示される
- **OK** → 既存データをすべて新しいデータで置き換え
- **キャンセル** → 既存データに追加（ID重複分はスキップ）

## データ構造

```json
{
  "id": "1710000000000",
  "company": "株式会社〇〇",
  "route": "エージェント",
  "agent": "リクルートエージェント",
  "media": "",
  "stage": "一次面接",
  "priority": "高（第一志望）",
  "dismissalReason": "",
  "dismissalNote": "",
  "interviewDate": "2026-03-20",
  "interviewer": "田中部長",
  "memo": "全体的な所感メモ",
  "events": [
    { "id": "e1", "date": "2026-03-01", "type": "応募",    "memo": "" },
    { "id": "e2", "date": "2026-03-07", "type": "カジュアル面談", "memo": "雰囲気良好。" },
    { "id": "e3", "date": "2026-03-14", "type": "一次面接", "memo": "手応えあり。" }
  ],
  "details": {
    "jobDescription": "募集要項のテキスト...",
    "companyUrl": "https://example.com",
    "attachments": [
      "https://example.com/jd.pdf",
      "https://example.com/slide.pdf"
    ]
  },
  "createdAt": "2026-03-01T10:00:00Z",
  "companyUnderstanding": {
    "事業":         { "summary": "", "points": [""], "tags": [], "highlights": [] },
    "課題特徴":     { "summary": "", "points": [""], "tags": [], "highlights": [] },
    "技術取り組み": { "summary": "", "points": [""], "tags": [], "highlights": [] },
    "カルチャー":   { "summary": "", "points": [""], "tags": [], "highlights": [] },
    "自分との接点": { "summary": "", "points": [""], "tags": [], "highlights": [] },
    "やりたいこと": { "summary": "", "points": [""], "tags": [], "highlights": [] }
  }
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | 一意なID（`Date.now()` で自動生成） |
| `company` | string | 企業名（必須） |
| `route` | string | 応募経路 |
| `agent` | string | エージェント名（route が「エージェント」の場合） |
| `media` | string | 媒体名（route が「その他」の場合） |
| `stage` | string | 現在の選考フェーズ |
| `priority` | string | 優先度ラベル（`"高（第一志望）"` / `"中（様子見）"` / `"低（優先度低）"` / `""`） |
| `dismissalReason` | string | お見送り理由（stage が「お見送り」の場合） |
| `dismissalNote` | string | お見送り自由記述 |
| `interviewDate` | string | 次回面接日（YYYY-MM-DD） |
| `interviewer` | string | 面接官名 |
| `memo` | string | 企業全体のメモ（HTML形式で保存） |
| `events` | array | 選考履歴イベント一覧（タイムライン） |
| `events[].id` | string | イベントID |
| `events[].date` | string | イベント日（YYYY-MM-DD） |
| `events[].type` | string | イベント種別（応募 / フェーズ名 / その他メモ） |
| `events[].memo` | string | イベントのメモ |
| `details` | object | 募集要項情報（省略時は自動補完） |
| `details.jobDescription` | string | 募集要項テキスト |
| `details.companyUrl` | string | 企業URL |
| `details.attachments` | string[] | 添付資料URLの配列 |
| `createdAt` | string | 作成日時（ISO 8601） |
| `companyUnderstanding` | object \| null | 企業理解データ（省略時は `null`、初回保存時に自動生成） |
| `companyUnderstanding[セクションキー]` | object | 各セクション（事業 / 課題特徴 / 技術取り組み / カルチャー / 自分との接点 / やりたいこと） |
| `companyUnderstanding[key].summary` | string | セクションの一言概要 |
| `companyUnderstanding[key].points` | string[] | 箇条書きポイント |
| `companyUnderstanding[key].tags` | string[] | タグ（`#` プレフィックス付き） |
| `companyUnderstanding[key].highlights` | number[] | ★ハイライト済みポイントのインデックス配列 |

> **後方互換性：** `details` / `priority` / `companyUnderstanding` フィールドが存在しない旧フォーマットのデータをインポートまたは読み込んだ場合、`normalizeLog()` が `LOG_DEFAULTS` をベースに自動補完します。新フィールドを追加した際も `LOG_DEFAULTS` に定義するだけで既存データへのフォールバックが保証されます。

## 選択肢

### 応募経路

| 経路 | 追加入力欄 |
|------|-----------|
| エージェント | エージェント名 |
| 直接応募 | なし |
| LinkedIn | なし |
| リファラル | なし |
| その他 | 媒体名（Wantedly / Green / ビズリーチ 等） |

### 選考フェーズ・バッジカラー

| フェーズ | バッジカラー | 追加入力欄 |
|---------|------------|-----------|
| カジュアル面談 | グレー | なし |
| 書類選考中 | ブルー（薄） | なし |
| 一次面接 | ブルー（濃） | なし |
| 二次面接 | パープル | なし |
| 最終面接 | オレンジ | なし |
| 内定 | エメラルドグリーン | なし |
| お見送り | レッド | お見送り理由・自由記述 |

### 優先度ラベル

| 優先度 | バッジカラー | 説明 |
|--------|------------|------|
| 高（第一志望） | レッド | 最優先で追いたい企業 |
| 中（様子見） | イエロー | 選考を続けながら判断する企業 |
| 低（優先度低） | グレー | 優先度が低い企業 |
| （未設定） | — | バッジ非表示 |

- 一覧カードに優先度バッジを表示（ステージバッジの左隣）
- 一覧画面の優先度フィルターで絞り込み可能
- 詳細画面サマリーカードのセレクトボックスで即時変更（保存不要）
- 企業フォームの編集画面でも設定可能

### お見送り理由（プリセット）

スキル不足 / カルチャーフィット / ポジションクローズ / 不明 / その他

### 閲覧モード（通常表示）

- 各イベントをカード形式で表示（日付・種別バッジ・メモを分離）
- メモは `whitespace-pre-wrap` で改行・段落をそのまま表示
- メモが120文字超または改行2行以上の場合、**3行で折りたたみ**表示
  - カード下部に「続きを読む ▼」ボタンが現れ、クリックで全文表示
  - 展開後は「▲ 閉じる」ボタンで折りたたみ
  - 複数イベントの折りたたみ状態は独立して管理

### 編集モード（編集時のみフォーム表示）

- 鉛筆アイコンをクリックした時のみインライン編集フォームが展開
- テキストエリアは最低5行・110px の高さを確保
- 入力に応じてテキストエリアが **自動伸長**（auto-resize）
- 保存/キャンセルで閲覧モードに戻る
- 日付を変更して保存すると日付順に自動ソート

### その他

- 左側の縦線＋丸ドットで時系列を視覚化
- 「選考を記録」ボタンからイベントを随時追記（メモ欄も同様にauto-resize対応）
- イベント追加時、ステージ系のイベントは現在フェーズを自動更新
- `お見送り` 記録時はお見送り理由もタイムラインに表示
- 各イベントの **ゴミ箱アイコン** をクリックするとインライン確認UI（削除 / キャンセル）を表示してから削除

## 変更履歴

### タイムラインUI/UX改善（2026-04-25）

**改善内容**
- **閲覧モードと編集モードを分離**
  - 通常時はカード形式のテキスト表示（フォームを常時表示しない）
  - 鉛筆アイコンをクリックした時のみ編集フォームを展開
- **タイムラインカードのデザイン刷新**
  - 各イベントを `bg-white rounded-xl border shadow-sm` のカードで囲み視認性を向上
  - カードヘッダー（日付・種別バッジ・操作ボタン）とメモ本文をセパレーターで分離
  - フォントサイズを `text-xs` → `text-sm` に拡大し読みやすさを改善
  - `whitespace-pre-wrap` により改行・段落をそのまま表示
- **メモの折りたたみ表示**（「続きを読む」トグル）
  - メモが120文字超 or 改行2行以上の場合、`line-clamp-3` で3行に折りたたみ
  - カード下部に「続きを読む ▼ / ▲ 閉じる」ボタンを表示
  - `expandedMemos`（`Set<id>`）state で各イベントの展開状態を独立管理
- **テキストエリアの改善**（新規記録フォーム・インライン編集フォーム共通）
  - 高さを `rows={3}` → `rows={5}` / `minHeight: 110px` に拡張
  - 入力に応じて高さが自動伸長（`scrollHeight` ベースの auto-resize）

**技術詳細**
- `DetailScreen` に `expandedMemos` state と `toggleMemo()` ヘルパーを追加
- タイムライン項目のビュー部分を`<div className="bg-white rounded-xl ...">` ラッパーで再設計
- `EventForm` / `EditEventForm` の textarea の `onChange` に `e.target.style.height` 更新処理を追加

### Markdownインポート機能追加（2026-04-24）

**新機能**
- **企業理解タブに「MDを取り込む」ボタンを追加**
  - ローカルの `.md` ファイルをブラウザの file input で選択して読み込む
  - FileReader でテキストを取得し、Markdown の `##` 見出しを基準にセクションへ自動マッピング
  - 見出しと企業理解セクションの対応：

    | Markdown見出し（部分一致）| 企業理解セクション |
    |--------------------------|-----------------|
    | `## 事業` | 事業 |
    | `## 課題` / `## 特徴` | 課題・特徴 |
    | `## 技術` / `## 取り組み` / `## 仕組み` | 技術・取り組み |
    | `## 組織` / `## カルチャー` / `## 文化` | カルチャー |
    | `## 自分との接点` / `## 接点` | 自分との接点 |
    | `## やりたいこと` / `## 志望動機` | やりたいこと |

  - Markdown の書式変換ルール：
    - `> blockquote` → 概要（一言）
    - `- リスト行` → ポイント
    - `tags: #タグ` 行 → タグ
  - 取り込み後も各フィールドは手入力で編集可能
  - 「保存する」を押すことで通常の企業理解データとして保存
  - 取り込み成功後3秒間「取り込み完了 ✓」のフィードバック表示
  - 既存の選考履歴・募集要項・メモタブには影響なし

**技術詳細**
- `MD_HEADING_MAP` 配列でMarkdown見出し → セクションキーの対応を一元管理
- `parseMarkdownToUnderstanding(mdText)` 関数でパース処理を実装
  - `##` 見出し行でセクションを分割し、`flushSection()` でデータを確定
  - 対応する見出しがない場合はそのセクションをスキップ（デフォルト値を保持）
- `fileInputRef` で `<input type="file">` を制御（同一ファイルの再選択に対応）
- mdファイル自体はリポジトリに含めない想定（個人用ローカルファイルをそのまま読み込み）
